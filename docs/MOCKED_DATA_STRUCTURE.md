# Mocked Data Structure

This document describes every mock data source in the AutoCap Group prototype and specifies what the real backend must implement to replace it. Generated on 2026-04-26 by `/mock-data-doc`.

## Summary

- **Total entities mocked:** 8 entities + 3 client-side state objects
- **Total endpoints to implement:** 15 API endpoints
- **Persistence strategy in prototype:** In-memory (resets on page reload) + localStorage for cookie consent only
- **Auth model in prototype:** None (no authentication required)
- **Form submission strategy:** Console logging + simulated delays (all forms are prototype-only)
- **Production CMS:** Strapi (headless CMS for content management)

## Prototype Architecture

This is a **frontend-only prototype** built with Next.js 15.5 using static generation. All data is:
- Hardcoded in TypeScript content files (`src/content/*.ts`)
- Statically generated at build time via `generateStaticParams()`
- No database, no API routes, no server-side mutations

The backend team will need to implement a full API layer to replace these static data sources.

## Production Architecture (with Strapi CMS)

**Backend stack:**
- **Strapi v5** (headless CMS) for content management
- **PostgreSQL** database (recommended for Strapi)
- **Next.js API routes** for custom endpoints (forms, downloads, etc.)
- **File storage:** Strapi media library → S3/CloudFlare R2

**Why Strapi:**
- **Content editing UI** out-of-the-box (no custom admin panel needed)
- **Auto-generated REST/GraphQL APIs** for content types
- **Media library** for team photos, news images, media kit assets
- **Role-based access** for content editors, admins
- **Content versioning** and draft/publish workflows
- **Reduces development time** by 60-70% vs. building custom CMS

**Which entities should be Strapi content types:**
- ✅ **Workshop** → Strapi collection
- ✅ **NewsArticle** → Strapi collection (with rich text component)
- ✅ **TeamMember** → Strapi collection
- ✅ **Testimonial** → Strapi collection
- ✅ **MediaAsset** → Strapi collection + media library
- ✅ **PrivacyPolicySection** → Strapi single type
- ✅ **StaticContent** → Strapi single types (homepage, about, etc.)
- ❌ **CookieConsent** → Client-side only (no backend needed)

**Which endpoints need custom implementation:**
- ❌ Contact forms (3) → Next.js API routes (not Strapi content types)
- ❌ File downloads → Custom endpoint with signed URLs
- ❌ Related articles logic → Custom endpoint or Strapi plugin

```typescript
interface Workshop {
```

### NewsArticle

**Shape:**
```typescript
interface NewsArticle {
  id: string;                    // String ID in format "news-NNN"
  title: string;                 // required, article headline
  excerpt: string;               // required, 1-2 sentence summary
  publishDate: string;           // required, ISO 8601 format "YYYY-MM-DD"
  author: string;                // required, author name or "AutoCap Group"
  category: 'Company News' | 'Press Release' | 'Industry Insights' | 'Media Coverage';
  imageUrl?: string;             // optional, hero image URL
  slug: string;                  // required, unique, URL-safe
  readTimeMinutes: number;       // required, estimated read time
  order: number;                 // required, for manual sorting (1-5 in mock)
  fullContent?: ArticleContentBlock[];  // optional, rich content array
  tags?: string[];               // optional, e.g., ["acquisition", "expansion"]
  relatedArticleIds?: string[];  // optional, manual article curation
}

// Rich content blocks for article body
type ArticleContentBlock =
  | { type: 'paragraph'; content: string }
  | { type: 'heading'; level: 2 | 3; content: string; id?: string }
  | { type: 'image'; src: string; alt: string; caption?: string; credit?: string }
  | { type: 'quote'; content: string; attribution?: string; role?: string }
  | { type: 'list'; style: 'bullet' | 'numbered'; items: string[] }
  | { type: 'callout'; variant: 'info' | 'highlight'; content: string }
```

- `src/components/news/NewsCard.tsx` — Article preview cards
- `src/components/news/ArticleBody.tsx` — Rich content renderer
- `src/components/news/RelatedArticles.tsx` — Related articles widget

**Mocked operations:**
| Operation | Mock function | Real endpoint (Strapi) | Notes |

| Filter by category | Client-side filter | `GET /api/articles?filters[category][$eq]=:category` | Strapi filter syntax |

## Mock-Specific Assumptions to Revisit

Things the prototype assumes that may not be true in production:

### Data Persistence
- ✅ **No database** — all data is hardcoded, resets on redeploy
- ✅ **No data mutations** — no create, update, or delete operations
- ✅ **No versioning** — content changes require code commits
- ❌ Production needs: PostgreSQL/MySQL database, migrations, seed scripts

### API Layer
- ✅ **No API routes** — prototype is static Next.js pages
- ✅ **No authentication** — all content is public
- ✅ **No rate limiting** — unlimited requests
- ✅ **No caching headers** — relies on Next.js static generation
- ❌ Production needs: RESTful API, auth middleware, rate limiting, CDN caching

### Form Submissions
- ✅ **Console logging only** — no emails sent, no data stored
- ✅ **No spam protection** — no CAPTCHA, no rate limiting
- ✅ **No email notifications** — success messages are fake
- ❌ Production needs: Email service (SendGrid/AWS SES), database storage, spam filtering

### File Uploads
- ✅ **No file storage** — media kit assets are placeholders
- ✅ **No image processing** — team photos are undefined
- ❌ Production needs: S3/CloudFlare R2, image optimization, CDN

### Search and Filtering
- ✅ **Client-side filtering** — all data fetched at once
- ✅ **No full-text search** — basic string matching only
- ❌ Production needs: Database indexes, full-text search (PostgreSQL or Elasticsearch)

### Pagination
- ✅ **No pagination** — all lists return all items
- ❌ Production needs: Cursor or offset pagination for news, workshops

### Analytics
- ✅ **No tracking** — cookie consent exists but doesn't load scripts
- ✅ **No server-side logging** — no request logs, no error tracking
- ❌ Production needs: Google Analytics, Sentry, server logs

### Timestamps
- ✅ **Client-generated timestamps** — cookie consent uses `Date.now()`
- ✅ **No created_at/updated_at** — entities lack audit trails
- ❌ Production needs: Server-side timestamps, audit logs
